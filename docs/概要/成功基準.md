# 成功の定義と制限事項

## 成功の定義

このプロジェクトが成功したと言えるのは以下の条件を満たす場合です。

### 1. 継続率

- ユーザーが**週3回以上**使う
- **3ヶ月以上**継続して使用

### 2. サイクルの実現

- 前回の課題が次回に繋がる
- 改善の実感がある
- 「前回→今回→次回」のループが回る

### 3. 気づきの深化

- 単なる記録ではなく、思考が深まる
- 「言語化できなかった感覚」が言葉になる
- AIの追加質問が有効に機能

### 4. 成長の実感

- 週次レビューで進捗を確認できる
- データが資産になっている
- 過去のメモを参照して学びが得られる

## 評価指標

### 定量的指標

| 指標 | 目標値 |
|------|--------|
| 週間利用日数 | 3日以上 |
| 月間メモ数 | 12件以上 |
| 週次レビュー確認率 | 80%以上 |
| 追記・振り返り率 | 20%以上 |

### 定性的指標

| 指標 | 確認方法 |
|------|---------|
| 思考の深化 | ユーザーヒアリング |
| 成長の実感 | 月次アンケート |
| 習慣化 | 利用ログの分析 |

## 制限事項

### 技術的制限

| 項目 | 制限 | 対策 |
|------|------|------|
| **Gemini API** | 無料枠: 1分あたり15リクエスト | リクエスト間隔を調整 |
| **Discord音声** | 最大25MB | 長い音声は分割を促す |
| **Discord画像/動画** | 無料: 25MB / Nitro: 500MB | Git LFSで管理 |
| **Git LFS** | GitHub無料枠: 1GB/月 | 容量監視、必要時に追加購入 |
| **Obsidian Dataview** | 10,000件以上で遅延可能性 | 個人利用なら問題なし |

### 運用上の注意

| 項目 | 説明 |
|------|------|
| **ユーザー数** | 1人想定（マルチユーザーはPhase 2以降） |
| **インターネット接続** | 必須（Gemini APIへのアクセス） |
| **タイムゾーン** | JST（日本標準時）前提 |
| **モバイル同期** | 追加設定が必要 |

### ファイルサイズ制限

| 入力タイプ | 制限 |
|-----------|------|
| 音声 | 25MB以下 |
| 画像 | 20MB以下 |
| 動画 | 20MB以下 |
| テキスト | 制限なし |

### 画像・動画の処理

**重要:** 画像・動画の自動解析は行いません。

- ファイル保存とユーザーコメントの記録のみ
- AIによる内容分析は将来の拡張として検討
- コスト効率とシンプルさを優先

## ストレージ容量の見積もり

### 月間使用量の想定

| 項目 | サイズ | 月30回記録時 |
|------|--------|-------------|
| 音声メモ（Markdown） | 2-5KB | 60-150KB |
| テキストメモ | 1-3KB | 30-90KB |
| 画像 | 1-5MB | 30-150MB |
| 動画 | 5-20MB | 150-600MB |

### Git LFS容量管理

```
月間最大: 約750MB
3ヶ月で: 約2.2GB
6ヶ月で: 約4.5GB

GitHub無料枠: 1GB/月のデータ転送
→ 3-6ヶ月で容量オーバーの可能性
→ 追加購入: $5/月で50GB
```

**対策:**
- 古いメディアファイルのアーカイブ
- 解像度・品質の調整
- 必要に応じて追加購入
