## 実装の注意点

### 段階的リリース

- 一度にすべてを実装しない
- Phase 1が安定してからPhase 2へ
- ユーザーフィードバックを重視

### データ移行

- Phase 1のデータ構造を後で変更しない
- 拡張可能な設計にする
- メタデータは柔軟に追加できるように

### パフォーマンス

- Gemini APIの呼び出し回数を最適化
- キャッシュを活用
- 非同期処理を活用

### エラーハンドリング

- Gemini APIエラー時のフォールバック
- ユーザーに分かりやすいエラーメッセージ
- ログを詳細に記録
