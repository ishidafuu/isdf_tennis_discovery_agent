# プロジェクトビジョン

## Tennis Discovery Agentとは

**テニスプレーヤーの「第2の脳」**

単なる記録アプリではなく、気づきを深化させ、継続的な成長をサポートするAIアシスタントです。

---

## 現状の課題

### Phase 1で実装済みの機能

- Discord音声メッセージ受信
- Gemini AIによる文字起こし
- 構造化データ抽出
- Obsidian Markdown生成
- GitHub自動Push

### 残っている課題

| 課題 | 説明 |
|------|------|
| **一律処理** | すべてが「気づきメモ」として同じ処理 |
| **シーン区別なし** | 壁打ち、スクール、試合の違いを認識しない |
| **継続性なし** | 前回→今回→次回のサイクルがない |
| **対話が浅い** | 記録して終わり、深掘りがない |

---

## 改善の方向性

### 3つのフローで整理

```
入力（Input） → 処理（Processing） → 出力（Output）
```

### 4つの設計原則

#### 1. Voice First（音声ファースト）

- 音声入力が基本、ボタンはオプション
- コート上でサッと記録できることが最優先
- 手がふさがっていても使える

#### 2. Cycle Oriented（サイクル志向）

- 前回→今回→次回の繋がりを重視
- 継続性がないと「記録アプリ」で終わる
- 課題の追跡と改善の実感

#### 3. Contextual Intelligence（文脈理解）

- シーンに応じた処理
- AIが空気を読んで振る舞いを変える
- 適切なタイミングで深掘り

#### 4. Obsidian-Only（シンプル設計）

- データベース不要、Markdownだけで管理
- バックアップも移行も簡単
- ツールロックインを避ける

## Discordチャンネル構成

```
Tennis Discovery Agent (サーバー名)

├── リアルタイム記録
│   ├── #壁打ち
│   ├── #スクール
│   ├── #試合
│   └── #フリー練習
│
├── 振り返り・分析
│   ├── #振り返り        ← 後日の追記・補足
│   ├── #週次レビュー    ← 週1の自動生成レビュー
│   ├── #質問           ← AIに質問（過去のメモ参照）
│   └── #分析           ← 成長分析
│
└── Bot管理
    └── #bot-commands   ← システムコマンド
```

---

## 2つの主要フロー

### フロー1: コート上（リアルタイム記録）

```
1. チャンネル選択
   #壁打ち / #スクール / #試合

2. 音声送信
   「今日のサーブ、トスを前に上げたらうまくいった」

3. Bot が自動処理
   - 文字起こし
   - シーン別に構造化
   - Markdown生成・保存

4. （オプション）ボタン表示
   [🤔 深堀質問] [📊 過去と比較] [🔖 重要マーク]

   ※ボタンは基本押さない
   ※「もっと何か欲しい」時だけ
```

### フロー2: 家で振り返り（後日の追記）

```
1. Obsidianでメモを読み返す
   「あ、これ追記したい」

2. #振り返り チャンネルへ
   「1/15のサーブメモに追記。実は〇〇も意識してた」

3. Bot が該当メモを特定
   - 日付で検索
   - 内容で検索

4. 追記セクションを追加
   元のMarkdownに追記を追加
```
